# AIppocampus.com Frontend Context

Audience: developers, agent builders, researchers, and curious users who need
to understand source-backed continuity quickly.

Visual thesis: warm, sourceful, and calm, but the first screen must still behave
like a developer product page. The site should feel trustworthy before it feels
poetic.

Content priority:

1. Help a visitor reach first recall quickly.
2. Explain source-backed continuity without claiming innate model memory.
3. Keep privacy and evidence boundaries visible.
4. Route deeper philosophy, field reports, and living discussion to stable
   pages or GitHub sources.

Design constraints:

- Use `site/assets/content.css` as the only stylesheet linked by HTML. It
  imports `components.css`.
- Keep navigation consistent across pages.
- Use self-hosted fonts and local assets only.
- Public pages must not include private memory artifacts, local machine paths,
  raw rollouts, registry data, sync bundles, tokens, cookies, or credentials.
